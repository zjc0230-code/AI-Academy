# 数据预处理技能

## 技能定位

数据预处理是数据分析和机器学习工作流程中的关键环节，其本质是将原始的、不完整的、充满噪声的数据转换为高质量的、可用于分析和建模的数据。在真实世界中，数据往往不是理想状态下的完美格式，而是存在缺失值、异常值、重复记录、格式不一致等各种问题。数据预处理技能的核心价值在于通过系统化的方法，识别并解决这些问题，为后续的数据分析和模型训练奠定坚实基础。

## 应用场景

### 商业数据分析
在电商运营场景中，用户行为数据可能存在以下问题：
- 用户ID缺失或格式不统一（如"U123"、"u-123"、"12345"）
- 订单金额包含异常值（如999999999的错误录入）
- 时间戳格式混乱（ISO8601、Unix时间戳、本地时间混合）

### 金融市场分析
股票行情数据的常见预处理需求：
- 处理停牌日产生的缺失值
- 去除除权除息引起的价格跳变
- 统一不同市场的交易时间格式

### 社交媒体分析
文本数据的预处理挑战：
- 去除HTML标签、特殊字符、表情符号
- 统一编码格式（UTF-8、GBK等）
- 处理多语言混合场景

## 核心能力

### 1. 数据质量评估
**问题识别能力**：
- 缺失值检测：识别哪些字段、多少比例的数据缺失
- 重复记录检测：识别完全重复或部分重复的记录
- 数据类型检查：验证字段是否与预期类型一致
- 数值范围检查：识别超出合理范围的异常值
- 格式一致性检查：如日期格式、电话号码格式等

**统计方法**：
- 缺失值比例统计（字段级别和整体级别）
- 数值型字段的描述性统计（均值、标准差、分位数）
- 分类字段的频次统计
- 相关性矩阵分析

### 2. 数据清洗

**缺失值处理策略**：
- **删除法**：当缺失比例较低时（如<5%），直接删除缺失记录
- **填充法**：用均值、中位数、众数填充数值型字段；用"未知"等标记填充分类字段
- **插值法**：基于时间序列或空间关系的插值方法
- **预测法**：使用机器学习模型预测缺失值（适用于重要字段）

**异常值处理策略**：
- **统计学方法**：基于3σ原则或IQR（四分位距）方法识别异常值
- **领域知识**：结合业务规则识别异常（如年龄<0或>120）
- **可视化方法**：通过箱线图、散点图等识别离群点
- **处理方式**：删除、修正、保留标记

**重复值处理**：
- 完全重复记录：保留第一条，删除其余
- 部分重复记录：结合业务规则判断合并或删除
- 去重原则：明确唯一键（如用户ID+时间戳）

### 3. 数据转换

**数据类型转换**：
- 字符串转数值：清理非数字字符后转换
- 字符串转日期：解析不同格式的时间戳
- 分类变量编码：
  - 标签编码：有序分类（如低、中、高→0、1、2）
  - 独热编码：无序分类（如北京、上海、广州→3个二进制特征）

**数据标准化**：
- **归一化**：将数值缩放到[0,1]范围，适用于不同量纲的特征
- **标准化**：转换为均值0、标准差1的分布，适用于算法要求正态分布的场景
- **对数变换**：处理右偏分布，减少长尾影响

**特征工程**：
- 数值型特征：创建聚合特征（如用户最近7天平均消费额）
- 时间型特征：提取年、月、日、星期、小时等维度
- 文本型特征：提取关键词、情感分数、文本长度等

### 4. 数据集成
当需要从多个数据源合并数据时：
- **键对齐**：确保不同数据源的主键一致（如用户ID格式统一）
- **字段映射**：处理字段名称不一致（如"user_name" vs "username"）
- **冲突解决**：当同一实体在不同数据源中信息不一致时，制定规则（最新优先、完整度优先等）
- **去重策略**：合并后再次检查重复记录

## 实践案例

### 电商用户行为分析案例

**问题描述**：
某电商平台的用户行为日志包含以下问题：
- 15%的记录缺少用户ID
- 5%的订单金额为负数（异常值）
- 20%的时间戳格式不一致
- 存在大量重复的点击记录

**处理步骤**：

1. **缺失值处理**：
```python
# 检查缺失值比例
missing_ratio = df['user_id'].isnull().mean()
print(f"缺失值比例: {missing_ratio:.2%}")

# 由于缺失比例较高，使用"未知"标记
df['user_id'] = df['user_id'].fillna('unknown')
```

2. **异常值处理**：
```python
# 识别负数金额
negative_orders = df[df['order_amount'] < 0]
print(f"负数订单数: {len(negative_orders)}")

# 删除异常记录
df = df[df['order_amount'] >= 0]
```

3. **时间戳统一**：
```python
# 统一转换为ISO8601格式
df['timestamp'] = pd.to_datetime(df['timestamp'], errors='coerce')
df['timestamp'] = df['timestamp'].dt.strftime('%Y-%m-%dT%H:%M:%S')
```

4. **去重处理**：
```python
# 去除同一用户在1分钟内的重复点击
df = df.drop_duplicates(subset=['user_id', 'timestamp'], keep='first')
```

### 股票价格预测案例

**问题描述**：
某股票历史数据存在以下问题：
- 停牌日产生缺失值
- 除权除息后价格出现跳变
- 部分交易日的成交量为0

**处理步骤**：

1. **缺失值插值**：
```python
# 使用前向填充和线性插值结合的方法
df['close'] = df['close'].fillna(method='ffill')
df['close'] = df['close'].interpolate()
```

2. **除权除息调整**：
```python
# 根据公告信息调整历史价格
# 假设某日除权比例为0.9
adjustment_factor = 0.9
df['close'] = df['close'] / adjustment_factor
```

3. **异常值标记**：
```python
# 标记成交量为0的交易日
df['is_suspension'] = df['volume'] == 0
```

## 最佳实践

### 1. 可重现性
- 记录所有的预处理步骤和参数
- 使用配置文件管理预处理规则
- 保存原始数据和处理后数据的对比

### 2. 文档化
- 记录每个处理步骤的原因和依据
- 注明数据源的获取时间和版本
- 维护数据字典和字段说明

### 3. 验证检查
- 处理前后对比数据质量指标
- 使用可视化方法验证处理效果
- 与业务专家确认处理结果的合理性

### 4. 性能考虑
- 对于大数据，使用分块处理或并行计算
- 避免在内存中保存过多中间结果
- 考虑使用增量处理策略

## 常见误区

### 1. 盲目删除缺失值
**误区**：看到缺失值就删除，导致样本量大幅减少
**正确做法**：先分析缺失原因，再决定删除、填充或标记

### 2. 忽略数据背后的业务逻辑
**误区**：仅基于统计学规则处理数据，忽略了业务合理性
**正确做法**：结合领域知识，如年龄异常、负数订单等需业务确认

### 3. 过度处理
**误区**：为了追求"完美"数据，进行过多的转换和修正
**正确做法**：在质量和信息量之间找到平衡，避免引入新的偏差

### 4. 忽视时间顺序
**误区**：在使用未来信息（如均值填充）处理过去数据
**正确做法**：对于时间序列数据，只能使用历史信息进行预处理
