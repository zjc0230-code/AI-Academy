# 数据清洗技能（SK-DA-001）

## 技能定义

数据清洗是数据分析流程中的关键环节，旨在识别、纠正或删除数据中的错误、不一致、重复或不完整的记录，以确保数据质量，为后续的数据分析和建模提供可靠的数据基础。

## 核心能力

### 1. 数据质量评估

**能力描述**：能够系统性评估数据集的质量状况，识别存在的数据问题。

**执行步骤**：
1. **数据完整性检查**
   - 统计每个字段的缺失值数量和占比
   - 识别关键字段（如ID、时间、核心指标）的缺失情况
   - 评估缺失值的分布特征（随机缺失、完全随机缺失、非随机缺失）

2. **数据一致性检查**
   - 检查同一字段的数据格式是否统一（日期格式、数值精度、字符串大小写等）
   - 验证关联字段之间的逻辑一致性（如：开始时间≤结束时间、销售额≥0等）
   - 检查重复记录（完全重复、部分重复）

3. **数据准确性检查**
   - 识别异常值（超出合理范围的数据点）
   - 验证业务规则约束（如：年龄在0-120岁之间、状态值为枚举值等）
   - 交叉验证数据源的一致性

4. **数据唯一性检查**
   - 检查主键字段的唯一性
   - 识别需要唯一约束的业务字段是否存在重复

**输出成果**：
- 数据质量评估报告（含问题列表、严重程度、影响范围）
- 数据质量问题可视化（缺失值热力图、异常值散点图等）

---

### 2. 缺失值处理

**能力描述**：根据缺失值的分布特征和业务背景，选择合适的缺失值处理策略。

**处理策略**：

**策略1：删除法**
- **适用场景**：缺失值占比<5%且随机缺失
- **删除方式**：
  - 删除含缺失值的整行记录（适用于行数据较多时）
  - 删除含缺失值的整个字段（适用于字段缺失率>30%时）
- **注意事项**：删除前需评估数据量和样本代表性

**策略2：填充法**
- **适用场景**：缺失值占比5%-30%或缺失值具有重要业务意义
- **填充方式**：
  - **数值型字段**：
    - 均值填充：适用于正态分布、无明显异常值的数据
    - 中位数填充：适用于偏态分布或存在异常值的数据
    - 众数填充：适用于离散型数值（如评分等级）
    - 分位数填充：使用P25/P75等分位数
    - 线性插值：适用于时间序列数据
  - **分类字段**：
    - 众数填充：使用出现频率最高的类别
    - 指定值填充：填充"Unknown"、"Other"等特定值
  - **逻辑填充**：
    - 基于相关字段推断：如根据年龄和性别推断职业
    - 基于规则填充：如根据产品类目填充默认价格

**策略3：建模预测**
- **适用场景**：缺失值占比30%-50%、字段重要性高、数据量大
- **预测方法**：
  - 回归模型：基于其他字段预测缺失值（适用于数值型字段）
  - 决策树/KNN：基于相似样本预测缺失值
  - 多重插补：生成多个可能的填充值，提高稳定性
- **注意事项**：需要充足的数据量和相关特征

**策略4：保留缺失状态**
- **适用场景**：缺失值本身包含信息（如"未填写"可能代表某种态度）
- **处理方式**：
  - 将缺失值标记为特定类别（如"Missing"、"NULL"）
  - 创建缺失指示变量（新字段标识是否缺失）

**决策流程**：
1. 分析缺失值占比（<5%→删除；5%-30%→填充；30%-50%→建模；>50%→删除字段）
2. 判断缺失类型（随机/非随机）
3. 评估字段重要性（核心字段→精心处理；次要字段→简单处理）
4. 选择最适合的填充方式并执行

### 3. 异常值处理

**能力描述**：识别并处理数据中的异常值，避免异常值对分析结果产生误导。

**异常值识别方法**：

**方法1：统计学方法**
- **Z-Score方法**：计算每个数据点的Z分数（数据点与均值的标准差倍数）
  - 阈值：|Z|>3视为异常值
  - 适用场景：数据近似正态分布
- **IQR方法**：基于四分位数识别异常值
  - 下限：Q1 - 1.5×IQR
  - 上限：Q3 + 1.5×IQR
  - 超出界限视为异常值
  - 适用场景：非正态分布数据
- **3σ原则**：数据点超出均值±3个标准差视为异常值

**方法2：可视化方法**
- **箱线图（Box Plot）**：直观显示数据分布和异常值
- **散点图**：识别离群点（适用于二维数据）
- **直方图**：观察数据分布是否有异常峰值或尾部

**方法3：业务规则方法**
- 基于业务经验设定合理范围
  - 如：年龄0-120岁、订单金额>0等
- 基于历史数据设定动态阈值
  - 如：订单金额超过历史平均值+3个标准差

**异常值处理策略**：

**策略1：验证修正**
- 适用场景：怀疑是数据录入错误或系统bug
- 处理方式：
  - 追溯原始数据源进行核实
  - 与业务方确认异常值的真实性
  - 如确认错误，修正为正确值

**策略2：删除**
- 适用场景：异常值数量少、无业务意义、对整体分析影响小
- 处理方式：直接删除异常记录

**策略3：截断（Winsorization）**
- 适用场景：异常值数量适中、需要保留数据量
- 处理方式：将异常值设置为上下限值
  - 如：将超过P99的数据点设置为P99值

**策略4：变换**
- 适用场景：异常值反映真实业务，但影响模型性能
- 处理方式：
  - 对数变换：适用于偏态分布、存在极大值的数据
  - Box-Cox变换：更灵活的幂变换方法
  - 归一化/标准化：降低异常值影响

**策略5：分箱**
- 适用场景：异常值较多、需要降低异常值影响
- 处理方式：将连续数值划分为离散区间
  - 如：将收入分为"低/中/高"三档

**决策流程**：
1. 使用多种方法识别异常值（统计学+可视化+业务规则）
2. 验证异常值的真实性（确认是错误还是真实业务情况）
3. 评估异常值的影响（对均值、方差、模型的影响程度）
4. 选择合适的处理策略

### 4. 重复数据处理

**能力描述**：识别并处理数据中的重复记录，避免重复数据影响分析结果。

**重复类型识别**：

**类型1：完全重复**
- 定义：所有字段的值都完全相同
- 识别方法：使用所有字段作为去重键

**类型2：部分重复**
- 定义：关键字段相同，但部分字段有差异
- 识别方法：基于业务主键或核心字段组合识别
- 如：用户ID相同但注册时间不同

**类型3：逻辑重复**
- 定义：不同字段记录的同一实体
- 识别方法：需要基于业务逻辑判断
- 如："张三"和"zhangsan"可能是同一个人

**处理策略**：

**策略1：删除重复**
- 适用场景：完全重复记录
- 保留规则：
  - 保留最新记录（基于时间戳）
  - 保留最完整记录（字段缺失最少）
  - 保留主记录（有明确主键标识）

**策略2：合并记录**
- 适用场景：部分重复或逻辑重复
- 合并规则：
  - 取最新值（基于时间戳）
  - 取非空值（优先保留有效数据）
  - 取最大/最小值（根据业务逻辑）
  - 取平均值（适用于数值型字段）

**策略3：标记重复**
- 适用场景：需要追溯数据来源或审计需求
- 处理方式：创建重复标识字段，标记重复记录

**注意事项**：
- 去重前需确认业务需求（某些场景需要保留历史记录）
- 保留原始数据备份
- 记录去重规则和结果（用于追溯）

### 5. 数据格式标准化

**能力描述**：统一数据格式，确保数据一致性和可分析性。

**标准化场景**：

**场景1：日期时间标准化**
- 统一格式：YYYY-MM-DD、YYYY-MM-DD HH:MM:SS
- 处理方式：
  - 解析多种日期格式（如"2025/03/25"、"Mar 25, 2025"）
  - 处理时区问题（统一转换为UTC或目标时区）
  - 提取时间特征（年、月、日、星期几、季度等）

**场景2：数值型字段标准化**
- 统一精度：保留合理的小数位数
- 单位统一：如"万元"转换为"元"、"kg"转换为"g"
- 处理方式：
  - 识别并转换单位（基于单位字段或业务规则）
  - 舍入处理（四舍五入、向上/向下取整）

**场景3：文本字段标准化**
- 大小写统一：统一为小写或大写
- 空格处理：去除首尾空格、将多个连续空格替换为单个空格
- 特殊字符处理：去除或替换特殊字符（如换行符、制表符）
- 去除冗余词：如去除"有限公司"、"Co., Ltd."等后缀

**场景4：分类字段标准化**
- 枚举值统一：将相似值映射到统一值
  - 如："男"/"M"/"Male"统一为"男"
- 类别合并：将细分类别合并为高级类别
  - 如："iPhone 12"/"iPhone 13"/"iPhone 14"合并为"iPhone"
- 创建分类字典：建立分类标准映射表

**执行流程**：
1. 分析当前数据格式（统计各字段的不同格式数量）
2. 制定标准化规则（基于业务需求和最佳实践）
3. 执行格式转换（使用函数或正则表达式）
4. 验证标准化结果（检查是否还有未处理的格式）

### 6. 数据逻辑验证

**能力描述**：基于业务规则验证数据的逻辑一致性，确保数据合理可信。

**验证维度**：

**维度1：字段内部验证**
- 验证规则：
  - 数值字段范围检查（如：年龄0-120、订单金额>0）
  - 枚举字段值检查（如：状态值只能是"Pending"/"Processing"/"Completed"）
  - 正则表达式验证（如：邮箱格式、手机号格式）

**维度2：字段间验证**
- 验证规则：
  - 逻辑关系检查：开始时间≤结束时间、折扣前价格≥折扣后价格
  - 相关性检查：销售额=单价×数量（考虑误差容忍度）
  - 状态流转验证：订单状态必须按流程流转

**维度3：跨表验证**
- 验证规则：
  - 外键一致性：子表的关联字段在主表中必须存在
  - 数据完整性：订单表中的商品ID必须在商品表中存在
  - 聚合一致性：订单总金额应等于所有订单商品金额之和

**维度4：时间序列验证**
- 验证规则：
  - 时间单调性：时间戳应该是递增的
  - 时间间隔合理性：相邻记录的时间间隔应在合理范围内
  - 时间周期性：周期性数据的周期应保持一致

**处理策略**：

**策略1：标记问题数据**
- 适用场景：问题数据占比小、需要人工审核
- 处理方式：创建问题标识字段，标记不符合规则的记录

**策略2：自动修正**
- 适用场景：问题明确、修正规则清晰
- 处理方式：
  - 基于规则修正（如：折扣后价格>折扣前价格时，交换两个值）
  - 基于关联字段修正（如：根据订单总金额和数量反推单价）

**策略3：回退默认值**
- 适用场景：无法确定正确值
- 处理方式：设置合理的默认值（如：状态默认为"Unknown"）

**验证报告**：
- 统计验证失败的数量和占比
- 按失败类型分类（字段内部/字段间/跨表/时间序列）
- 按严重程度分级（致命错误/严重错误/警告）
- 提供问题样例和修正建议

## 执行流程

### 阶段1：数据探索（15%时间）
1. 加载数据集，查看数据结构（字段类型、样本数据）
2. 统计数据基本情况（行数、列数、内存占用）
3. 检查各字段的缺失值、唯一值、数据类型
4. 生成数据概览报告

### 阶段2：问题识别（25%时间）
1. 执行数据质量评估（完整性、一致性、准确性、唯一性）
2. 识别缺失值问题（数量、占比、分布）
3. 识别异常值问题（使用统计学和可视化方法）
4. 识别重复数据问题（完全重复、部分重复、逻辑重复）
5. 识别格式问题（日期、数值、文本、分类）
6. 识别逻辑问题（字段内部、字段间、跨表、时间序列）

### 阶段3：制定方案（15%时间）
1. 为每类问题制定处理策略
2. 评估策略的影响（数据量、质量、业务逻辑）
3. 确定执行顺序（优先处理影响大的问题）
4. 准备备份数据（处理前备份原始数据）

### 阶段4：执行清洗（35%时间）
1. 处理缺失值（选择合适的填充/删除策略）
2. 处理异常值（验证/删除/截断/变换/分箱）
3. 处理重复数据（删除/合并/标记）
4. 标准化数据格式（日期、数值、文本、分类）
5. 验证数据逻辑（字段内部、字段间、跨表、时间序列）
6. 记录每步处理结果（处理前后的对比）

### 阶段5：质量验证（10%时间）
1. 对比清洗前后的数据质量指标
2. 验证清洗结果（缺失值减少、异常值处理、重复消除）
3. 检查是否引入新问题
4. 生成清洗报告（处理过程、结果、建议）

## 最佳实践

### 实践1：保留原始数据
- 清洗前备份原始数据
- 使用版本控制（Git）跟踪数据变化
- 记录每步操作的详细日志

### 实践2：迭代式清洗
- 不要一次性处理所有问题
- 先处理明显问题，验证效果后再处理复杂问题
- 每次处理后检查是否引入新问题

### 实践3：文档化处理规则
- 记录所有清洗规则和决策依据
- 创建清洗脚本（可重复执行）
- 编写清洗报告（便于追溯和审计）

### 实践4：与业务方确认
- 对模糊不清的问题，与业务方确认处理方式
- 确认异常值是错误还是真实业务情况
- 确认字段的业务含义和约束条件

### 实践5：验证清洗效果
- 对比清洗前后的数据质量指标
- 检查数据分布是否合理
- 在后续分析中观察结果是否改善

### 实践6：考虑业务场景
- 不同的业务场景对数据质量的要求不同
- 探索性分析可容忍一定脏数据
- 模型训练需要高质量数据
- 报表展示需要准确一致的数据

## 常见问题与解决方案

### 问题1：缺失值过多（占比>50%）
**解决方案**：
- 考虑删除该字段（如果字段不重要）
- 使用高级建模方法（如多重插补）
- 将缺失值作为新的特征（缺失指示变量）

### 问题2：异常值难以判断
**解决方案**：
- 使用多种方法交叉验证（统计学+可视化+业务规则）
- 与业务方确认异常值的真实性
- 保留原始数据，同时创建处理后的副本

### 问题3：重复数据难以合并
**解决方案**：
- 明确合并规则（优先保留哪些字段）
- 创建合并日志，记录合并依据
- 对不确定的重复数据，保留所有记录并标记

### 问题4：数据格式不统一
**解决方案**：
- 建立数据字典，定义标准格式
- 使用正则表达式解析多种格式
- 创建格式转换映射表

### 问题5：清洗后数据丢失太多
**解决方案**：
- 评估清洗策略是否过于严格
- 使用更温和的处理方式（如保留而非删除）
- 收集更多原始数据补充

## 评估指标

### 定量指标
- **缺失率**：清洗前/后缺失值的占比
- **重复率**：清洗前/后重复记录的占比
- **异常值占比**：清洗前/后异常值的数量
- **数据完整性**：关键字段的完整记录占比
- **数据一致性**：符合业务规则的记录占比
- **数据保留率**：清洗后保留的数据量占比

### 定性指标
- **数据可信度**：业务方对数据的信任程度
