**版本**：v1.0
**更新日期**：2026-06-26

---

## 技能元数据结构

每个技能必须包含以下字段：

### 1. 定义（Definition）

**格式**：清晰、简洁的技能定义

**示例**：
```
代码规范遵循：遵循业界标准编写清晰、可维护的代码
```

---

### 2. 核心要求（Core Requirements）

**格式**：具体的、可衡量的要求

**示例**：
```
- 命名规范（变量、函数、类名）
- 注释规范（函数注释、复杂逻辑说明）
- 格式化统一（缩进、空格、换行）
```

---

### 3. 适用场景（Use Cases）

**格式**：该技能适用的场景列表

**示例**：
```
- 新代码开发
- 代码审查
- 代码重构
```

---

### 4. 验收标准（Acceptance Criteria）

**格式**：可验证的验收标准

**示例**：
```
- 通过 lint 工具检查
- 代码可读性评分 ≥ 8/10
- 无明显命名或格式问题
```

---

## 技能等级评估

### 入门级（Entry）

- 能理解技能的基本概念
- 能在指导下完成简单任务
- 缺乏独立应用能力

**示例**：能编写基本代码，但缺乏规范意识

---

### 熟练级（Proficient）

- 能独立完成技能相关的任务
- 能处理常见问题和边界情况
- 能指导初级人员

**示例**：能独立完成模块开发，遵循规范，编写测试

---

### 专家级（Expert）

- 能解决复杂问题
- 能优化流程和工具
- 能制定标准和规范

**示例**：能指导他人，制定团队规范，优化开发流程

---

## 示例：完整技能定义

### 技能名称

**代码规范遵循**

### 完整定义

```markdown
### 代码规范遵循

**定义**：遵循业界标准（如 Google Java Style、PEP 8、ESLint）编写清晰、可维护的代码。

**核心要求**：
- 命名规范（变量、函数、类名）
- 注释规范（函数注释、复杂逻辑说明）
- 格式化统一（缩进、空格、换行）
- 避免魔法数字和硬编码
- 代码复杂度控制（圈复杂度 < 10）

**适用场景**：
- 新代码开发
- 代码审查
- 代码重构

**验收标准**：
- 通过 lint 工具检查
- 代码可读性评分 ≥ 8/10
- 无明显命名或格式问题
```
