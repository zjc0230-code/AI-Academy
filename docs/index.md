## 🚀 快速开始

### 开发者快速开始

#### 1. 开发一个新技能

1. **阅读规范**: 阅读 [技能元数据规范](./spec/skill-metadata.md)
2. **创建技能目录**: 在 `skills/` 下创建 `{skill-id}/` 目录
3. **编写元数据**: 创建 `skill.yaml` 文件，填写技能元数据
4. **实现功能**: 在 `src/` 目录下实现技能代码
5. **编写测试**: 在 `tests/` 目录下编写测试用例
6. **编写文档**: 在 `docs/` 目录下编写使用文档
7. **提交审核**: 提交 PR 进行审核

#### 2. 开发一个新专业

1. **阅读规范**: 阅读 [专业分类白皮书](./spec/major-taxonomy.md)
2. **确定专业代码**: 根据专业分类体系确定专业代码（如 OA-EM）
3. **创建专业目录**: 在 `majors/` 下创建 `{专业代码}/` 目录
4. **编写专业元数据**: 创建 `major.yaml` 文件
5. **收集技能**: 收集或开发专业所需的技能
6. **编写文档**: 编写专业的使用文档和示例
7. **提交审核**: 提交 PR 进行审核

### 使用者快速开始

#### 1. 使用一个技能

1. **查找技能**: 通过关键词、分类或语义检索找到需要的技能
2. **阅读文档**: 查看技能的 `README.md` 和 `docs/quickstart.md`
3. **安装依赖**: 根据 `requirements.txt` 安装依赖
4. **配置参数**: 根据 `config/default.yaml` 配置参数
5. **运行示例**: 参考 `examples/` 目录下的示例代码
6. **集成使用**: 将技能集成到你的 Agent 中

#### 2. 使用一个专业

1. **查找专业**: 根据需求找到对应的专业
2. **阅读文档**: 查看专业的 `README.md` 和 `docs/quickstart.md`
3. **完成前置配置**: 根据 `major.yaml` 中的 `prerequisites` 完成配置
4. **安装技能**: 安装专业所需的所有技能
5. **测试使用**: 参考 `examples/` 目录下的示例
6. **部署上线**: 将专业部署到你的 Agent

## 🤝 贡献指南

### 如何贡献

AI Academy 欢迎所有形式的贡献：

1. **提交新技能**: 开发新的技能并提交审核
2. **提交新专业**: 创建新的专业能力包
3. **优化规范**: 提出规范改进建议
4. **修复问题**: 修复文档或代码中的问题
5. **反馈意见**: 提供使用反馈和建议

### 贡献流程

1. Fork AI Academy 仓库
2. 创建分支（`git checkout -b feature/xxx`）
3. 进行修改
4. 提交更改（`git commit -m 'Add some feature'`）
5. 推送分支（`git push origin feature/xxx`）
6. 提交 Pull Request

### 代码规范

- 遵循 PEP 8 代码风格
- 添加必要的注释和文档
- 确保所有测试通过
- 更新相关文档
