## 执行策略

### 迭代原则
1. **数据优先**：确保 Web 数据与文档完全同步
2. **逐步推进**：按阶段逐步完成迭代，每阶段完成后测试
3. **质量保证**：每完成一个功能点，确保链接正确、样式美观
4. **持续优化**：根据用户反馈持续优化

### 开发流程
1. 读取 docs 文档，获取最新专业信息
2. 更新/创建专业页面
3. 更新首页和专业列表页面数据
4. 测试所有链接和功能
5. 提交代码到 Git 仓库
6. 推送到远程仓库

### 定期同步
- 每周检查一次 docs 文档更新
- 发现更新立即同步到 Web 页面
- 确保内容一致性
