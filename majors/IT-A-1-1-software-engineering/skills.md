## 技能列表

### 1. 代码规范遵循

**定义**：遵循业界标准（如 Google Java Style、PEP 8、ESLint）编写清晰、可维护的代码。

**核心要求**：
- 命名规范（变量、函数、类名）
- 注释规范（函数注释、复杂逻辑说明）
- 格式化统一（缩进、空格、换行）
- 避免魔法数字和硬编码
- 代码复杂度控制（圈复杂度 < 10）

**适用场景**：
- 新代码开发
- 代码审查
- 代码重构

**验收标准**：
- 通过 lint 工具检查
- 代码可读性评分 ≥ 8/10
- 无明显命名或格式问题

---

### 2. 单元测试编写

**定义**：使用测试框架（JUnit、pytest、Jest）为代码编写自动化测试用例。

**核心要求**：
- 测试覆盖率 ≥ 80%
- 覆盖边界条件（null、空值、异常输入）
- 测试命名清晰（describe/it/should）
- 使用 Mock 隔离依赖

**适用场景**：
- 新功能开发
- 代码重构
- Bug 修复验证

**验收标准**：
- 所有测试通过
- 覆盖率报告无红色
- 关键路径测试覆盖

---

### 3. Git 工作流规范

**定义**：遵循 Git Flow 或 GitHub Flow 进行版本控制和代码协作。

**核心要求**：
- 分支命名规范（feature/、bugfix/、hotfix/）
- Commit 信息规范（type: description）
- Pull Request 审查流程
- 代码合并前自测

**适用场景**：
- 多人协作开发
- 版本发布
- Bug 修复

**验收标准**：
- PR 标签完整（review、approved）
- CI/CD 流水线通过
- 无遗留的 TODO 或 FIXME

---

## 技能组合示例

**场景**：开发一个用户登录接口

1. **代码规范遵循**：编写符合命名和格式规范的代码
2. **单元测试编写**：为登录接口编写测试用例（包括成功、失败、异常情况）
3. **Git 工作流规范**：创建 feature 分支、提交代码、发起 PR

## 能力评估

- **入门**：能编写基本代码，但缺乏规范意识
- **熟练**：能独立完成模块开发，遵循规范，编写测试
- **专家**：能指导他人，制定团队规范，优化开发流程
