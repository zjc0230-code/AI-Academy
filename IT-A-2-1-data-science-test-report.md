### 步骤4：Agent 执行任务（当前表现）

**预期输出（基于当前skills.md）**：

```python
import pandas as pd
import matplotlib.pyplot as plt

# 读取数据
df = pd.read_csv('sales_data.csv')

# 基本分析
print(df.describe())

# 简单可视化
plt.plot(df['date'], df['sales'])
plt.show()
```

**问题分析**：
- ❌ 缺少数据清洗步骤（处理缺失值、异常值）
- ❌ 分析维度单一（没有按产品类别、地区分组）
- ❌ 可视化图表简陋（没有标题、标签、配色）
- ❌ 缺少结论和建议
- ❌ 代码可读性差（没有注释、变量命名不规范）
- ❌ 缺少最佳实践（没有使用 Seaborn、没有保存图表）

## 🚨 发现的问题

### 问题1：执行细则层缺失

**现象**：
- Agent 知道"要做什么"（数据分析、可视化）
- 但不知道"怎么做"（具体步骤、方法、规范）

**影响**：
- 输出代码质量不稳定
- 分析维度不全面
- 可视化效果差

**根本原因**：
- skills.md 中只有一句话描述
- 缺少具体的数据分析工作流
- 缺少代码模板和最佳实践

---

### 问题2：可视化规范缺失

**现象**：
- Agent 不知道如何制作专业级可视化
- 不知道图表类型选择、配色、布局等规范

**影响**：
- 可视化效果不专业
- 无法有效传达数据洞察

**根本原因**：
- 没有可视化设计指南
- 没有优秀案例参考

---

### 问题3：数据分析方法论缺失

**现象**：
- Agent 不知道标准的数据分析流程
- 不知道如何选择合适的分析方法

**影响**：
- 分析可能遗漏关键洞察
- 缺少系统性的分析框架

**根本原因**：
- 没有数据分析方法论文档
- 没有常见场景的解决方案

---

### 问题4：工具集成规范缺失

**现象**：
- Agent 不知道如何使用各类数据科学工具
- 不知道工具的最佳实践

**影响**：
- 可能选择不合适的工具
- 代码效率低

**根本原因**：
- 没有工具使用指南
- 没有代码模板库

## 📝 测试总结

### 核心发现
1. **数据科学专业与小红书专业问题类似**：技能定义缺少执行细则层
2. **需要建立三层文档体系**：
   - 层级1：是什么（skills.md - 已有）
   - 层级2：怎么做（guides/ + quick-reference.md - 需创建）
   - 层级3：做成什么样（examples/ - 需创建）

3. **Agent 学习路径应统一**：
   - 读取 skills.md（获取概览）
   - 自动发现详细参考链接
   - 读取详细指南（获取方法）
   - 查看示例（对标优秀）
   - 执行任务（产出结果）
